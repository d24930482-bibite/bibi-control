Read / write / iterate distinctions

Gene access (M4)

Note: gene("X") was removed in M4. Use these instead:

  • b.genes["X"] — loud KeyError on miss (use when the gene must exist)
  • b.genes.get("X") or b.genes.get("X", default) — tolerant, returns None/default on miss
  • b.genes["X"] = v — write (stages a mutation)
  • for g in b.genes — iterate all genes; len(b.genes)
  • Gene handle: g.name, g.type, g.value

Collection branches

A spanning collection (e.g. workspace.bibites, world.nodes) is read-only aggregate-only: only count / sum / mean / median / min / max / quantile / where / group_by are available — no set, set_expr, or delete.

A single-save collection (e.g. s.bibites from world.open()) also exposes set / set_expr / delete. Mutation is per-save by construction; call s.commit(path) to write back.

Which save(s) you read follows from where you start: s = world.open() → the current head save (one working partition, the only writable scope); world.bibites → that world's whole retained history (every committed revision); workspace.bibites → all worlds' history. Both spanning scopes read committed history only — never an uncommitted working save — and expose world_id / sim_time (the per-revision time axis) as columns.

workspace.nodes asymmetry (M1)

workspace.nodes is NOT the brain-node aggregate. It is the host process-node listing (the cluster node registry). To query brain nodes across all worlds use world.nodes (per world) or reach them via workspace.synapses / workspace.genes. See workspace/automation.go:88-91 for the authoritative comment.

Brain-nav attrs (M5)

On a synapse element (b.synapses[i]): syn.source, syn.target → the source/target node ArrayElement.

On a node element (b.nodes[i]): n.inputs, n.outputs → read-only sub-collection of incoming/outgoing synapses.

These nav attrs are conditional: source/target only on synapses; inputs/outputs only on nodes.

Settings

  • save.settings.simulation["SettingName"] — loud read
  • save.settings.simulation.get("SettingName") — tolerant read
  • save.settings.simulation["SettingName"].set(value) — staged write
  • save.settings.material("MatName")["SettingName"] — material scope

Data-derived surfaces (per save-format version)

The per-entity column list (b.attr names on an Entity/ ArrayElement) comes from the generated metadata that tracks the Bibites save format. It changes between Bibites versions; use dir(b) in the notebook to see the current live column list. The cheatlist does not freeze this list (see the save-format-churn policy).

Copy-paste recipes

Open a world and iterate bibites

w = workspace.world("My World")
s = w.open()
for b in s.bibites:
    print(b.desc, b.genes["Diet"].value)
s.commit("/path/to/output.zip")

Bulk mutation with .where().set()

s = workspace.world("My World").open()
n = s.bibites.where("health < 0.5").set("energy", 1.0)
print(n, "bibites updated")
s.commit("/path/to/output.zip")

Spanning aggregate (all worlds)

# All bibites across all worlds' history.
result = workspace.bibites.where("dead = false").count()
print(result)

# Per-world breakdown.
for row in workspace.bibites.group_by("world_id").count():
    print(row)

Per-world brain stats (M1/M5)

# All-worlds node aggregate via world.nodes (NOT workspace.nodes).
w = workspace.world("My World")
for node in w.nodes.where("node_type = 'sigmoid'"):
    print(node)

# Synapse nav (M5).
s = w.open()
for b in s.bibites:
    for syn in b.synapses:
        src = syn.source   # the source node ArrayElement
        tgt = syn.target   # the target node ArrayElement
        print(src.node_desc, "->", tgt.node_desc, "weight:", syn.weight)

Gene keyed-lookup (M4)

s = workspace.world("My World").open()
for b in s.bibites:
    diet = b.genes.get("Diet")       # tolerant: None on miss
    if diet is not None:
        b.genes["Diet"] = diet.value * 1.1  # loud write
s.commit("/path/to/output.zip")

start_node + wait

w = workspace.add_world("/path/to/save.zip", "run1")
nd = workspace.start_node(world_id=w.id, sim_steps=1000)
nd.wait()
print(nd.status)

transfer

src = workspace.world("Source").open()
dst = workspace.world("Dest").open()
sel = src.bibites.where("health > 0.8")
workspace.transfer(sel, dst)
dst.commit("/path/to/dest_out.zip")

query (power-user SQL escape hatch)

# Raw DuckDB SQL — all tables are accessible.
rows = workspace.query("SELECT world_id, count(*) FROM bibites GROUP BY world_id")
for row in rows:
    print(row)

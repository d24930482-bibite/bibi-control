## Correctness against the plan

1. Registry CRUD — all required methods present and consistent with the existing
   struct/Input + re-Get-by-id pattern: CreateWorkspace/GetWorkspace/ListWorkspaces;
   CreateWorld/GetWorld/ListWorlds/SetWorldHead/RevisionsForWorld;
   CreateNode/GetNode/ListNodes/BindNode/SetNodeStatus. IDs allocated with
   `uuid.NewString()`. `nodes.world_id` mapped NULL<->"" via `sql.NullString`.
2. Parent threading — `WorldID` added to `RevisionInput`/`Revision`, carried through
   `normalizeRevision`, persisted in the INSERT via `sql.NullString` (empty -> NULL,
   non-empty must satisfy the FK). The INSERT column list lives in exactly one place
   (`insertRevisionTx`, taking a `sqlExec` interface satisfied by both `*sql.DB` and
   `*sql.Tx`), used by both `RecordRevision` and `RecordRevisionAdvancingHead` — matches
   the plan's "one INSERT site" requirement.
3. `RecordRevisionAdvancingHead` — single `BeginTx` wrapping insert + head UPDATE,
   `defer tx.Rollback()` + checked `Commit`, `input.WorldID` forced to `worldID`,
   re-read via `RevisionByID` after commit. tier/blob_present/refcount/mirror columns
   omitted from INSERT so schema defaults apply ('full'/1/0/NULL).
4. Tier fields scanned in all three SELECTs (RevisionByID, RevisionsBySHA256, and
   RevisionsForWorld) through the shared `scanRevisionRows` extension; world_id via
   NullString, mirror_schema_version via NullInt64, blob_present via int->bool.
5. Refcount/tier (folded-in G1): IncBlobRef / DecBlobRef are per-content-hash
   (`WHERE sha256 = ?`), Dec floored with `AND refcount > 0` so the schema
   `CHECK (refcount >= 0)` can never trip. EvictRevisionBlob runs in a tx, refuses a
   world head (`ErrRevisionIsHead`) and refcount>1 (`ErrBlobStillReferenced`), flips
   tier='mirror_only'/blob_present=0, never deletes bytes or the row. UnreferencedBlobs
   reports distinct refcount=0 hashes not held by any blob_present=1 row.
   PromoteRevision is idempotent (`WHERE id=? AND tier='mirror_only'`, 0 rows on
   already-full is not an error). Eviction ref-accounting convention documented in a
   comment.
6. Typed sentinels `ErrRevisionIsHead` / `ErrBlobStillReferenced` added; no
   `ErrNotRematerializable` (correctly left to G4). `EvictRevisionBlob` on a missing
   revision returns raw `sql.ErrNoRows`, so the existing `IsNotFound` helper still works.

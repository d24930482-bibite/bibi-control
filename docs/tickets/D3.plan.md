# D3 — reload: ship head→drop path + RELOAD; IngestAutosave → append+advance head

**Tier: high — persistence + monotonic head advance (RecordRevisionAdvancingHead), crash safety of the ship-head→drop reload path (partial failure must not lose data or strand a node), dual-key mirror import, parent threading; D3 is a dependency of E1 (fan-out). `--model-exec opus --model-review opus`.**

## Goal

Add the two node ship/receive methods to the `workspace` package, both as Go methods on
`*Workspace` (no Starlark yet — that is E1):

- `ReloadNode(ctx, nodeID)` — materialize the bound world's **head** blob to the node's
  persisted `drop_path` and send `simctl.RELOAD` so the game loads the new save (workspace→game).
- `IngestAutosave(ctx, nodeID, path)` — stabilize and parse a game autosave file, **dedup by
  content hash** against the world's current head (skip if unchanged), then `blobstore.Put` +
  `RecordRevisionAdvancingHead(parent=head)` + dual-key DuckDB import so the world head advances
  to the ingested save (game→workspace). `path == ""` defaults to the node's live
  `InfoResult.LastAutosave.Path`.

"Done" = both methods land in a **new file** `workspace/node_reload.go` with table-driven tests
in `workspace/node_reload_test.go`, reusing the C1/C3 commit/import primitives verbatim (no
re-derived refcount, no second `IncBlobRef`), the existing D2 node-control fake, and the C1/C3
world fixtures; `go test ./workspace/...` and `go test ./...` stay green.

- **NEW `workspace/node_reload_test.go`** — tests (see Tests).

  `saveparser/thebibites`, and the existing `workspace/*.go` already expose every primitive D3

## Approach

### `nodeRowByLogicalID(ctx, nodeID)` helper (unexported, in node_reload.go)
The active set (`w.nodes`) is keyed by logical node id → `*noderuntime.Runtime`, but the
binding (`world_id`) and `drop_path` live only on the persisted `nodes` row. There is no
existing public getter keyed by logical id (`GetNode` keys on the PK). Reuse the exact
resolution pattern at `node.go:304-315`: call `w.store().ListNodes(ctx, w.id)`, scan for the row
whose `NodeID == nodeID`, return it; error "no persisted row for logical node %q" on miss.
Do **not** hold `w.mu` for the ListNodes read (consistent with the D2 passthrough methods).

### `ReloadNode(ctx, nodeID)` (workspace→game ship-head)
1. Nil/`nodeID==""` guards (match the existing method preamble style).
2. `rt, ok := w.Node(nodeID)` (node.go:188; takes/releases `w.mu` for the map peek). On miss:
   `"workspace: ReloadNode: unknown node %q"`. Resolving the runtime first means we never write a
   drop file for a node we cannot then command to reload.
3. `node, err := w.nodeRowByLogicalID(ctx, nodeID)`. Require `node.WorldID != ""` (a node with no
   bound world cannot be reloaded — fail loudly) and `node.DropPath != ""` (no destination — fail
   loudly: `"workspace: ReloadNode: node %q has no drop_path"`).
4. Resolve the bound world's **head** blob bytes. Take `w.mu` only for the registry+blob reads
   (these are reads, not the single-writer mutating path, but keeping them consistent with `Load`
   is fine; **do not** call `w.Load`/`OpenWorld` — reload ships the persisted head bytes, not a
   possibly-staged working copy). Steps, mirroring `working_set.go:38-72`:
   - `world := w.store().GetWorld(ctx, node.WorldID)`; `IsNotFound` → clean error.
   - `world.HeadRevisionID == nil` → error (`"world %q has no head to reload"`).
   - `rev := w.store().RevisionByID(ctx, *world.HeadRevisionID)`.
   - **Blob-present guard (G4 seam):** `if !rev.BlobPresent { return plain error "head revision %d
     is mirror_only (blob evicted): cannot reload" }`. Add the comment
     `// G4: replace with ErrNotRematerializable` exactly as working_set.go:63 does, so G4 has one
     uniform site to upgrade.
   - `data := w.blobs().Get(ctx, rev.BlobRef)`.
5. **Write the drop file crash-safely:** write to `node.DropPath + ".tmp"` (or
   `os.CreateTemp(filepath.Dir(dropPath), ...)`), `f.Sync()`, `f.Close()`, then
   `os.Rename(tmp, dropPath)` — an atomic replace so the game never reads a half-written save, and
   a crash mid-write leaves the prior drop file intact (no data loss; the head is still durable in
   the blobstore regardless). `os.MkdirAll(filepath.Dir(dropPath), 0o755)` first.
6. Send RELOAD **after** the file is durably in place: `res, err := simctl.New(rt).Reload(ctx)`
   (simctl.go:59). Run the IPC round-trip **without holding `w.mu`** (match D2 NodeInfo at
   node_control.go:50). Wrap errors `"workspace: ReloadNode %q: %w"`.
7. Return `(ipc.ReloadResult, error)` so callers/E1 can surface `res.Save`/`res.Ok`. Ordering
   rationale (write→fsync→rename→RELOAD): a RELOAD that arrives before the bytes are durable would
   make the game load stale/partial data; reversing the order is the subtle bug a test guards.

### `IngestAutosave(ctx, nodeID, path)` (game→workspace append)
1. Nil/`nodeID==""` guards. `rt, ok := w.Node(nodeID)`; miss → unknown-node error.
2. `node, err := w.nodeRowByLogicalID(ctx, nodeID)`; require `node.WorldID != ""`.
3. **Default the path** when `path == ""`: `info := simctl.New(rt).Info(ctx)` (node_control.go:50
   pattern); if `info.LastAutosave == nil || info.LastAutosave.Path == ""` → error
   `"workspace: IngestAutosave %q: node has no autosave to ingest"`; else `path = info.LastAutosave.Path`.
   IPC round-trip runs without `w.mu`.
4. **Stabilize the file** before parsing (architecture-doc requirement: never parse a file the
   game is still writing). Implement a small `waitForStableFile(ctx, path)` local helper: `os.Stat`
   in a loop, comparing `(Size, ModTime)` between successive polls separated by a short interval
   (e.g. 100ms), returning once two consecutive polls match (or after N stable polls); bound by
   `ctx` and a max-attempts cap, returning a clear error on timeout or a never-appearing file. Keep
   the interval/attempts as named consts so the test can rely on a short settle. (Foreground
   `sleep` is fine inside a normal Go test; the harness `sleep` restriction is about the Bash tool,
   not Go code.)
5. Parse: `archive, err := tb.ParseFile(path, nil)` (parse_archive.go:13). `archive.SHA256`
   (archive.go:51) is the whole-file content digest.
6. **Dedup by content hash against the current head** (skip an unchanged autosave):
   - `world := w.store().GetWorld(ctx, node.WorldID)`.
   - If `world.HeadRevisionID != nil`: `head := w.store().RevisionByID(ctx, *world.HeadRevisionID)`;
     `if head.SHA256 == archive.SHA256 { return <zero Revision>, false-ish/no-op, nil }`.
     Return signature: `(revisionstore.Revision, bool, error)` where the `bool` = "ingested"
     (false on dedup-skip). This mirrors CommitWorld's no-op contract (commit.go:87) — a skipped
     ingest moves no head and writes no mirror.
7. **Append + advance head + dual-key import** under `w.mu` (single DuckDB writer). This is the
   `importWorldFromArchive` body (world.go:80-159) adapted for a non-first revision — **reuse its
   exact structure**, do not re-derive:
   - Read the file bytes: `data, err := os.ReadFile(path)` (the parser is path-only; reuse the
     same bytes for the blob, matching AddWorld at world.go:34). Re-acquire/confirm `archive.SHA256`
     is the digest the blob will hash to — `blobs().Put(ctx, data)` returns `ref`; assert
     `ref.SHA256 == archive.SHA256` and fail loudly on divergence (world.go:98-102 pattern; guards
     the file changing between stabilize and read).
   - `RecordScriptRun` with `Status: "ingested"`, `ScriptSHA256: ref.SHA256` (world.go:108 pattern).
   - Derive `simTime` from `archive.Scene` (world.go:121-125 pattern).
   - **Re-read the head inside the lock** as the parent (TOCTOU guard, exactly as commit.go:67-77):
     `parentID := world.HeadRevisionID` re-fetched under `w.mu`.
   - `rev, err := w.store().RecordRevisionAdvancingHead(ctx, node.WorldID, simTime,
     revisionstore.RevisionInput{ParentID: parentID, WorldID: node.WorldID, SourcePath: path,
     BlobRef: ref, ScriptRunID: run.ID})`. **No separate `IncBlobRef`** — the recorder self-refs
     the blob in-tx (world.go:127-139 comment, store.go:380). A second increment would break G2
     eviction.
   - **Dual-key DuckDB import** (world.go:141-158 / commit.go:118-142): history partition keyed by
     `ref.SHA256` via `duckdb.ReplaceExtractedSave` (additive, never disturbs prior revisions);
     working partition keyed by `node.WorldID` via `duckdb.ReplaceExtractedSave`. Use
     `tb.ExtractTables(key, archive)` directly — **ingest needs no `reparseCommitted`** because the
     parsed `archive` here is fresh from disk and its typed projections are accurate (unlike the
     post-`Session.Apply` archive in CommitWorld, commit.go:99-114). Call out this distinction in a
     comment so the executor does not copy reparseCommitted.
   - **Drop the cached working copy:** `delete(w.worlds, node.WorldID)` under `w.mu` so a later
     `OpenWorld` lazy-reloads from the now-advanced head (mirrors commit.go:150). Required because
     an already-loaded `LoadedSave` would otherwise be stale at the prior head.
8. Return `(rev, true, nil)`.

### Lock discipline (load-bearing)
`w.mu` is a non-reentrant `sync.Mutex` (workspace.go:45). Never call `w.Node`, `w.OpenWorld`,
`w.Load`, or `nodeRowByLogicalID` while holding `w.mu` — they each take it themselves and would
self-deadlock. The IPC round-trips (Reload/Info) and file IO run **outside** `w.mu`; only the
registry-record + dual-key DuckDB import in IngestAutosave runs **inside** `w.mu`. This matches
the established split in commit.go (OpenWorld before the lock; record+import inside) and
node_control.go (IPC outside the lock).

## Risks / seams the executor must not break

- **No second `IncBlobRef`.** `RecordRevisionAdvancingHead` self-refs the blob in-tx
  (store.go:337-384). The C1/C3 invariant: exactly one ref per revision. A manual `IncBlobRef`
  after recording would set `Refcount == 2` and strand the blob from G2 eviction. Test #4 asserts
  `Refcount == 1`.
- **Dual-key discipline.** History partition is keyed by the new revision **sha256** via
  `ReplaceExtractedSave` (the delete is a no-op for a never-seen sha256, so prior history survives
  — immutability). Working partition is keyed by the **world id**. Confusing the two corrupts or
  loses history; test #4(c) guards retention.
- **Ingest does NOT reparse.** Unlike CommitWorld (which reparses because `Session.Apply` leaves
  typed projections stale, commit.go:99-114), the ingest archive is fresh from `ParseFile`; its
  projections are accurate. Do not copy `reparseCommitted` — extract from the already-parsed
  archive.
- **Crash-safe reload ordering:** write→fsync→atomic-rename the drop file, **then** RELOAD. A
  RELOAD before durable bytes loads stale/partial data; a crash mid-write must not clobber the
  prior drop file or the head (head stays in the blobstore regardless).
- **Monotonic head + atomic parent threading:** the insert-revision + SetWorldHead are one SQLite
  tx inside `RecordRevisionAdvancingHead`; re-read the head **inside `w.mu`** as the parent
  (TOCTOU guard, commit.go:64-77) so a concurrent ingest/commit cannot steal the parent.
- **Stabilize before parse:** never parse a file the game is still writing (architecture doc,
  workspace_plan.md:530). Bound the settle loop by ctx; fail loudly on a vanishing file.
- **Non-reentrant `w.mu`:** resolve runtime (`w.Node`) and persisted row (`nodeRowByLogicalID`,
  which calls `ListNodes`) and run all IPC/file IO **outside** the lock; only the
  record+dual-key-import critical section holds it.
- **Parallel-ticket overlap (flag for the herder):** G2 (planning, eviction → likely
  `workspace/eviction.go` + `revisionstore` evict methods) and F1 (reviewing,
  `savemutator/thebibites/` + a `workspace` transfer file) touch the same `Workspace` struct but
  **different files**. D3 adds only `workspace/node_reload.go` + its test and **edits no existing
  file** (the `nodeRowByLogicalID` helper is new and self-contained; node.go is left untouched).
  No serialization needed unless G2's executor also chooses to add a `nodeRowByLogicalID`-style
  helper — if so, the herder should let whichever lands first own it. The G4 blob-present guard in
  ReloadNode is an additive comment-marked seam (`// G4: replace with ErrNotRematerializable`)
  identical to working_set.go:63, so G4 upgrades it uniformly later.

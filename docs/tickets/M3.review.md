  the `gene` vs `genes` miss asymmetry (documented as deliberate, M4 resolves by deletion).

- The implementation matches the plan at every named site.

## Contract verification

1. **Fold at boundary only / canonical case never mutated.** `foldLookup` returns the
   backing index; every caller dereferences the *stored* row (`set.backing[idx]`,
   `backing[idx]`) or the actual `World` struct. No lowercased string is ever written into
   `GeneName` / `World.Name` / `SettingName` / `OwnerID`, nor passed to any `SQLValueRef`
   mirror locator. Confirmed.
2. **Exact wins cheaply (rule 3).** `foldLookup` has an exact-key fast path before the fold
   scan; `worldByName` checks exact matches first (len 1 → return; >1 → loud) before the
   fold scan. A genuine `m`/`M` coexistence stays individually addressable by exact name.
   Confirmed by the exact-case assertions in all three collision tests.
3. **Deterministic loud collision.** `foldLookup` collects all distinct case-folding keys,
   sorts them, and returns an ambiguity error naming the colliding canonical keys when ≥2.
   `worldByName`'s fold branch keeps its 0/1/many switch (many → ambiguous error). The
   error message is order-independent.
4. **Miss asymmetry UNCHANGED (only the match folded).** `GeneCollection.Get` still returns
   `found=false` on a miss (→ KeyError, loud); `geneBuiltin` still returns `starlark.None`
   on a miss (silent). Only an *ambiguity* is surfaced as an error from the builtin (a
   real surprise, not an absent gene). Doc comments at both sites updated to state the split
   is deliberate. Confirmed.
5. **`settingRow` 3-return refactor preserves the owner-id sentinel handling.** Simulation
   /independent scopes use exact sentinel lookup (`"settings"`/`"independents"`, no fold);
   material (and zone) owner ids are user-typed and are case-folded with a loud
   owner-ambiguity error. The out-of-range backing guard is preserved (returns
   `(nil,false,nil)`, distinct from the ambiguity `err`). The sole caller
   (`SettingScope.Get`) is updated to the new signature; grep confirms no other caller.
6. **Column scope** bounded exactly as planned: `resolveColumn` lowercases `col` before the
   `attrRegistry()[kind][...]` read; no fold over the generated registry, no `Attr` dispatch
   folding.

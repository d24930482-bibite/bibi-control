# E1 — Workspace automation Starlark globals

Source: `docs/workspace_plan.md#e1` (Track E). Deps B3, C3, C4, D3, G4 all merged.

## Goal

Ship a **new, effectful, host-trusted** Starlark binding layer that exposes the
already-built `workspace.Workspace` Go surface to scripts, so an operator can automate the
loop (worlds, nodes, info/ingest/reload, read-only queries) from a single hermetic
Starlark invocation. "Done" = a new file `workspace/automation.go` (package `workspace`)
that builds a `starlark.StringDict` whose root `workspace` global is a `*Workspace`-backed
object exposing the E1-scoped surface; `world` and `node` handles returned from it expose
their E1-scoped methods; every binding maps to a concrete Track C/D/G Go method already on
`*Workspace`; every Go error (notably `ErrNotRematerializable` from G4) reaches the script
as a clean Starlark error (never a panic); and a host entry-point `RunAutomation(ctx, ws,
program, opts)` runs the program via the existing `script.Run` engine with that dict. E1
does **not** implement E2's head-advancing `world.open()`/`s.commit()` semantics, transfer
(F2), or gc (G3) — those bind to methods that do not yet exist and are explicitly deferred.

**Deferred — DO NOT bind in E1** (no backing Go method exists; binding would have to stub):
- `world.open()` returning a head-bound `Save` whose `s.commit()` advances the head — **E2**.
  (`workspace.open(path)` ad-hoc/ephemeral is also E2's `open()` story; E1 ships none of the
  `open`/`Save`-from-world surface.)
- `workspace.transfer(src,dst,…)` — **F2** (`w.Transfer` does not exist).
- `workspace.gc()` — **G3** (`w.GCUnreferencedBlobs` does not exist).

If a script calls a deferred name it must get a normal Starlark "no such attribute" error
(the default from returning `(nil, nil)` in `Attr`), not a binding. Keeping these absent is
how E1/E2 do not overlap on the same `Attr` switch later.

## Files to change

- **NEW `workspace/automation.go`** (package `workspace`) — the entire binding layer. Holds:
  - `RunAutomation(ctx context.Context, ws *Workspace, program []byte, opts script.Options) (script.Result, error)` — host entry-point; builds the globals dict and calls `script.Run` (engine.go:36).
  - `AutomationGlobals(ws *Workspace) starlark.StringDict` — returns `{"workspace": &workspaceValue{ws: ws}}` (mirrors `thebibites.Globals` shape, bindings.go:10, but effectful root object instead of `open`/aggregates).
  - `type workspaceValue struct{ ws *Workspace }` — `starlark.Value` + `HasAttrs`; `Attr` switch dispatches the `workspace.*` builtins/handles above.
  - `type worldValue struct{ ws *Workspace; world revisionstore.World }` — `starlark.Value` + `HasAttrs`; carries an immutable snapshot of the `World` row (id/name/head/sim_time) and the `*Workspace` for method dispatch.
  - `type nodeValue struct{ ws *Workspace; node revisionstore.Node }` — same shape for nodes.
  - small conversion helpers (`mapsToStarlark`, `infoToDict`, `evictResultToDict`, etc.) — see Approach step 6.
- **NEW `workspace/automation_test.go`** (package `workspace`) — Starlark-level tests (see Tests). Reuses the existing helpers in `workspace/world_test.go` (`newWorkspace`, `fixturePath`, `fixtureA`/`fixtureB`) and the node fake pattern from `workspace/node_control_test.go` / `workspace/node_reload_test.go`.

**No other file is edited.** In particular `script/thebibites/*` is **not touched** by E1
(the herd flagged it a serialization point: B3 + C2/C3/C4 + F2 contend there). E1 lives
entirely in package `workspace`, reusing `script.Run` and `script.Options` from package
`script`. This keeps E1 off the `script/thebibites` contention seam entirely.

## Approach

1. **Root globals + entry-point.** `AutomationGlobals(ws)` returns a one-key dict
   `{"workspace": &workspaceValue{ws}}` (no `open`, no `save` — E2 owns Save delivery; no
   host aggregates — those are the sandboxed save DSL's, not the effectful surface; honor
   the trust-boundary rule in workspace_plan.md:533). `RunAutomation` calls
   `script.Run(ctx, program, AutomationGlobals(ws), opts)` and returns its `script.Result` /
   error. The engine already recovers builtin panics into clean `RunError`s (engine.go:57),
   so a Go panic inside a binding cannot escape — but bindings still return errors, never
   panic, for clean messages.

2. **`workspaceValue` Attr dispatch.** Implement `starlark.Value` (String/Type/Freeze/Truth/Hash
   — copy the unhashable-Hash pattern from `Save` in save_value.go:22-26) and `HasAttrs`.
   `Attr(name)` returns `starlark.NewBuiltin(name, fn)` for methods and direct values for
   fields. `AttrNames()` lists exactly the in-scope names. Each builtin uses
   `starlark.UnpackArgs` (the established arg pattern, e.g. bindings.go:30) and pulls a
   `context.Context` via the thread — **decision: thread a `context.Context` into the bindings
   by capturing it in a closure** at `AutomationGlobals`/`RunAutomation` time (the engine's
   `starlark.Thread` carries no ctx; `RunAutomation` has the ctx). Concretely:
   `AutomationGlobals` takes the ctx (or `workspaceValue` carries it) so each builtin calls the
   Go method with the run's ctx. Carry the ctx on `workspaceValue`/`worldValue`/`nodeValue`
   (a per-run value graph; safe because these values are constructed per `RunAutomation` call).

3. **World/node handle construction.** `workspace.worlds()` lists rows then wraps each in a
   `worldValue`. `workspace.world(id)` does `GetWorld`; on `revisionstore.IsNotFound(err)`
   return a Starlark error `"world %q not found"`. `worldValue.Attr` for `id/name` reads the
   snapshot; `head` returns the `*int64` head as a Starlark Int or `None`; `sim_time` returns
   the `*float64` as Float or `None`. Mutating handle methods (evict/load/unload) re-resolve
   through `w.*` methods (which take their own locks), not the stale snapshot.

4. **Node handle.** `workspace.nodes()` → `PersistedNodes` → wrap each row. `workspace.node(id)`
   resolves the persisted row by logical id (scan `PersistedNodes`) and wraps it; return a
   clean error if absent. `node.info/state/stop/resume/reload/ingest_autosave/kill` delegate
   to the matching `w.Node*` method keyed by `node.NodeID`. `ingest_autosave(path=None)`:
   `path` defaults to empty string → Go method defaults from `InfoResult.LastAutosave.Path`
   (node_reload.go:225). Return `{"ingested": bool, "revision_id": int, "sha256": str}` from
   the `(rev, ingested, err)` triple; on a dedup-skip (`ingested==false`) revision_id is 0.

5. **`world.query(sql)` — working copy.** There is no `*Workspace`-level working-copy query
   method; the working-copy read is `ls.query` on the open `LoadedSave` (sql.go:45), which is
   **unexported** and in package `thebibites`. **Decision: bind `world.query` to the open
   world's `Save.sql` via the public surface** — i.e. `w.OpenWorld(ctx, worldID)` returns the
   `*thebibites.LoadedSave`, but `thebibites` exposes the query only through the `Save` object's
   `sql` attr (save_value.go:35). E1 cannot reach `ls.query` (unexported) and must NOT add a
   method to `thebibites` (serialization-point rule). **Therefore: implement `world.query` by
   reusing the workspace-level read path scoped to the world** — bind it to
   `w.HistoryQuery(ctx, worldID, sql)` is WRONG (that is history, not working copy). The clean
   E1 choice: bind `world.query` to a thin new **read-only working-copy query on `*Workspace`**
   is also out (would touch new Go surface beyond bindings). **Resolution (chosen): E1 does
   NOT ship `world.query` (working-copy read); it ships `world.history_query` (C4 public
   method) only, and documents `world.query` as E2's job** (E2 owns the `world.open()` →
   `Save` object whose `s.sql` is the working-copy read). This keeps E1 strictly a binding of
   *existing public `*Workspace` methods* and avoids touching `thebibites`. Update the scope
   table: `world.query` moves to **deferred (E2)**; `world.history_query` stays in E1.
   *(Executor: do not invent a working-copy query path; bind only `HistoryQuery` for worlds and
   `Query` for the workspace.)*

6. **Conversions (reuse, do not reinvent).** Query results come back from `w.Query`/
   `w.HistoryQuery` as `[]map[string]any` (query.go:410 `scanRowsToMaps`). Convert each value
   with the same scalar mapping `script/thebibites/convert.go:fromSQLValue` uses — but that is
   unexported in `thebibites`. **Write a local `sqlScalarToStarlark(any) (starlark.Value, error)`
   in automation.go** mirroring `fromSQLValue` (nil→None, bool, string, int64, uint64, float64,
   `*big.Int`→`MakeBigInt`; reuse `duckdb.NormalizeSQLScanValue` as fromSQLValue does,
   convert.go:45). Map a `[]map[string]any` to a `*starlark.List` of `*starlark.Dict`. For the
   typed result structs (`ipc.InfoResult`, `StopResult`, `ResumeResult`, `ReloadResult`,
   `NodeState`, `EvictResult`) build `*starlark.Dict`s field-by-field with stable snake_case
   keys (e.g. info → `{tps, real_tps, paused, sim_time, last_autosave:{path,name,modified_unix,time}|None}`
   per ipc/commands.go:46; evict → `{candidates, demoted, bytes_deleted, refused_head, refused_shared}`
   per eviction.go:59). Keys are load-bearing for tests; name them once here.

7. **Error surfacing (G4).** Every binding returns `(nil, err)` on a Go error; Starlark wraps
   it into an `*EvalError` with a backtrace and `script.Run` normalizes it (engine.go:128).
   No special handling is needed for `ErrNotRematerializable` *value*, but the binding MUST NOT
   swallow or panic: e.g. `world.evict_history` then `node.reload()` on the evicted head returns
   the wrapped `ErrNotRematerializable` (node_reload.go:124) straight through as the builtin's
   error — assert the message reaches the script in tests. Same for `ensureReadOnly` rejections
   (query.go:255): a non-SELECT in `workspace.query`/`world.history_query` returns
   `ErrReadOnlyQuery` as a clean Starlark error.

8. **Trust boundary + non-looping doc.** Add a package/file doc comment stating: this surface
   is effectful/host-trusted (does IO/IPC), is separate from the sandboxed save DSL
   (`thebibites.Globals`), a single `RunAutomation` is one hermetic cycle (Starlark is
   non-looping), and the host re-invokes on a timer/event (workspace_plan.md:430-432).

## Tests (`workspace/automation_test.go`, package `workspace`)

Run: `go test ./workspace/...` (and `go test ./...` for the whole-suite gate). Each test
builds a workspace with `newWorkspace(t, ctx)` and a real fixture via `fixturePath(t,
fixtureA)` / `fixtureB`; node tests reuse the `net.Pipe` fake from `node_control_test.go`
(`newFakeNode`) and `node_reload_test.go` (`newReloadFakeNode`, configurable autosave path).

1. **`TestAutomation_AddAndListWorlds`** — program calls `workspace.add_world(path, "w1")`
   then `workspace.worlds()`; assert one world, `world.id` non-empty, `world.name == "w1"`,
   `world.head` is an int, `world.sim_time` is a float-or-None. Print results and assert via
   `result.Output`, OR (cleaner) have the program `workspace.add_world` and the test then
   asserts via Go (`ListWorlds`) — prefer **assert-via-Go-side state** for structural facts and
   **assert-via-print/Output** for value round-trips.
2. **`TestAutomation_WorkspaceQueryReadOnly`** — add a world, then program runs
   `workspace.query("SELECT count(*) AS n FROM bibites")`; assert the returned list/dict reaches
   the script (print the count, check Output). Second program runs
   `workspace.query("DELETE FROM bibites")` and asserts the run **fails** with the read-only
   message (assert `err != nil` and message contains "read-only" / "forbidden keyword").
3. **`TestAutomation_WorldHistoryQuery`** — add a world; `world.history_query(
   "SELECT count(*) AS n FROM world_saves")` returns ≥1 (the CTE is prepended by HistoryQuery);
   assert via Output.
4. **`TestAutomation_NodeInfoAndControl`** — register a `newFakeNode`, program calls
   `workspace.node(id).info()`, `.stop()`, `.resume(2.0)`; assert the returned dicts carry the
   fake's telemetry (`tps`, `last_autosave.name`, `previous_time_scale`, `time_scale`).
5. **`TestAutomation_NodeReloadAndIngest`** — `newReloadFakeNode` bound to a world with an
   autosave path pointing at `fixtureB`; program calls `node.ingest_autosave()` then
   `node.reload()`; assert (Go-side) head advanced after ingest and the drop file was written;
   assert the ingest dict reports `ingested == true`. Add a re-ingest of the same head proving
   `ingested == false` (dedup, node_reload.go:265).
6. **`TestAutomation_EvictHistory`** — add a world, commit/ingest a 2nd revision so there is a
   non-head revision, then program `world.evict_history(keep_last=1)`; assert the returned dict
   (`demoted >= 1`) and Go-side that the evicted revision is `mirror_only`.
7. **`TestAutomation_ErrNotRematerializable_SurfacesCleanly`** — force the head `mirror_only`
   with the existing `forceHeadMirrorOnly` helper (node_reload_test.go), bind a node, program
   calls `node.reload()`; assert the run errors and the message contains
   "mirror_only" / "cannot be rematerialized" (G4 sentinel reaches the script, no panic).
8. **`TestAutomation_WorldNotFound`** — `workspace.world("nope")` → run fails with a clean
   "not found" error (no panic).
9. **`TestAutomation_DeferredNamesAbsent`** — `world.open()` / `workspace.transfer(...)` /
   `workspace.gc()` each produce a normal Starlark "has no .open/.transfer/.gc attribute"
   error (proves the E1/E2/F2/G3 boundary is structural).

## Risks / seams (executor must not break)

- **Do NOT touch `script/thebibites/`.** It is a serialization point (B3 + C-track + F2
  contend). E1 binds only *public `*Workspace` methods* and reuses `script.Run`/`script.Options`
  from package `script` and the unexported-mirror conversion is **re-implemented locally** in
  `automation.go` (do not export `fromSQLValue` from `thebibites`).
- **Crisp E2 boundary:** ship NO `open()`/`Save`-from-world and NO working-copy `world.query`
  in E1. `world.open()` and `s.commit()` advancing the head are E2; binding a stub now would
  collide with E2's `Attr` switch. `world.history_query` (C4, history) and `workspace.query`
  (C4, whole-workspace) are the only query bindings E1 ships.
- **No transfer / no gc** — `w.Transfer`/`w.GCUnreferencedBlobs` do not exist (F2/G3); do not
  invent them or stub the bindings.
- **ctx threading:** Starlark threads carry no `context.Context`. Capture the run ctx in the
  value graph (`workspaceValue`/`worldValue`/`nodeValue` built per `RunAutomation` call) so each
  binding passes the run ctx to the Go method. Do not use `context.Background()` inside bindings
  (that would drop cancellation/timeouts the engine relies on).
- **Locking:** bindings call public `*Workspace` methods that each manage `w.mu` themselves
  (non-reentrant). Bindings must never hold any lock across calls and never reach `w.mu`/`w.duck()`
  directly — go through the public methods so the single-writer discipline (commit.go:31,
  working_set.go:160) is preserved.
- **Dict key stability:** the snake_case keys in step 6 are the test contract — name them once
  and keep them; a typo silently breaks the automation API for E2's example cycle.
- **Errors faithful, never panic:** return `(nil, err)` from every builtin on failure; rely on
  the engine's normalization. Verify `ErrNotRematerializable` and `ErrReadOnlyQuery` reach the
  script as messages (tests 2 & 7).

## Tier

Tier: **medium → `--model-exec sonnet`, `--model-review opus`.** Signals scanned:
- **Shared contract — yes** (new Starlark automation surface; E2's example cycle and future
  callers depend on the binding names + dict-key shapes) and **fan-out — yes** (E2 depends on
  E1, and E2 layers onto the same `Attr` switch). These two would push toward high, BUT:
- **Persistence/data-shape — no, concurrency/atomicity — no, security — no, subtle-invariant —
  no, novel-logic — no.** E1 is *additive, pattern-following wiring*: every binding is a thin
  passthrough to an already-built, already-tested `*Workspace` method; the dual-key / head /
  refcount / crash-safe invariants all live in C3/D3/G2 (already merged and tested), not in E1.
  The riskiest E1-local choice is "don't bind deferred names / don't touch `thebibites`", which
  is a boundary discipline a strong reviewer catches, not an algorithmic correctness hazard.
So: low-risk *execution* (a sonnet executor can follow this plan's explicit binding table), but
the shared-contract + fan-out means a sonnet mistake (wrong key name, a stubbed E2 binding,
swallowed error) is best caught by an **opus reviewer**. That is exactly the medium mapping.

**Perf flag: NONE.** Perf signals scanned: `world.history_query`/`workspace.query` call
`refreshMirrorCatalog` which is O(worlds × revisions) per query (the C4 N+1 already flagged and
*owned by C4*, not introduced by E1). E1 adds no loop over growing data, no N+1 round-trip, no
hot-path allocation of its own — it is a per-invocation binding layer over methods whose perf
characteristics are fixed upstream. No new perf concern is introduced, so no perf-review flag.

# G4 — Guardrails: ErrNotRematerializable on blob-dependent paths

## Goal

G2 can demote a revision to `mirror_only` (bytes deleted from the blobstore, catalog
row kept, `blob_present=0`). "Done" for G4 is: every workspace code path that needs a
revision's blob bytes checks `blob_present` first and, when the bytes are gone, fails
**loud with a single typed sentinel** — `workspace.ErrNotRematerializable` — instead of
returning an ad-hoc plain `fmt.Errorf` (today) or, worse, falling through to a
blobstore `Get` that errors opaquely. There is **no** mirror→blob fallback (that
non-rematerialization is structural per the spec). The typed error must be
`errors.Is`-matchable by callers (E1 will surface it as a clean Starlark error) and
must wrap enough context (world id / revision id) to be actionable in a log line.

Scope is exactly the two pre-marked guard sites (see "Files to change"); the audit
below proves no third blob-dependent path exists in the current tree.

## Audit — every blob-dependent path (grep `G4` + `blob_present` + `blobs().Get`)

The tree was searched for `blobs().Get`, `BlobPresent`, `RevisionByID`, `OpenWorld`,
`.Load(`. The blob-byte consumers are:

1. **`workspace/working_set.go` `Load`** (line ~64 guard, line ~69 `blobs().Get`) —
   materializes a world head to a temp file. This is the guard for `Load`,
   `OpenWorld` (delegates to `Load`), and `CommitWorld` (calls `OpenWorld` →
   `commit.go:53`). **GUARD SITE 1.** Pre-marked `// G4: replace with ErrNotRematerializable`.

2. **`workspace/node_reload.go` `headBlobBytes`** (line ~124 guard, line ~127
   `blobs().Get`) — resolves head bytes for `ReloadNode`. **GUARD SITE 2.** Pre-marked
   `// G4: replace with ErrNotRematerializable`.

3. **`workspace/commit.go` `reparseCommitted`** (line 167 `blobs().Get`) — reads the
   blob that was **just written in the same commit transaction**. That revision is
   `full`/`blob_present=1` by construction (it was recorded micro-seconds earlier in
   the same call), so it is **NOT a G4 site**. Do not add a guard here; doing so would
   require a redundant `RevisionByID` round-trip on the hot commit path for a revision
   that cannot be mirror_only.

4. **`savemutator/thebibites/transfer.go` (F1)** — its header comment states it
   "never touches a revision store ... F1 stops at" the Session boundary. It operates
   on already-opened `*Session` objects; the blob read that produced those Sessions
   happens upstream in `Load`/`OpenWorld` (site 1), which is already guarded. **NOT a
   G4 site.**

5. No package outside the worktree imports `bibicontrol/workspace` yet (E1 is the
   downstream consumer). There is no export path in v1 (out of scope per spec).

Conclusion: **exactly two guard sites**, both already pre-marked. G4 is a typed-error
swap at known locations, not a discovery exercise.

## Approach

1. **Define the sentinel** in `workspace/errors.go`:

   ```go
   package workspace

   import (
       "errors"
       "fmt"
   )

   // ErrNotRematerializable is returned by any blob-dependent path (Load/OpenWorld,
   // ReloadNode) when the target revision has been evicted to mirror_only
   // (blob_present=0): its bytes are gone and there is NO mirror→blob fallback
   // (non-rematerialization is structural, workspace_plan.md#g4). Callers match it
   // with errors.Is; E1 surfaces it as a clean Starlark error.
   var ErrNotRematerializable = errors.New("workspace: revision is mirror_only (blob evicted) and cannot be rematerialized")
   ```

   Add a small constructor that wraps the sentinel with the actionable context the
   current plain errors already carry (world id, revision id), so `errors.Is` still
   matches via `%w`:

   ```go
   func notRematerializable(worldID string, revID int64, op string) error {
       return fmt.Errorf("workspace: %s world %q head revision %d is mirror_only (blob evicted): %w",
           op, worldID, revID, ErrNotRematerializable)
   }
   ```

   Confirm `rev.ID`'s type against `revisionstore.Revision` when wiring the signature
   (it is the SQLite row id; match the existing `%d` formatting at the two sites).

2. **Site 1 — `working_set.go` `Load`.** Replace:
   ```go
   // G4: replace with ErrNotRematerializable
   if !rev.BlobPresent {
       return nil, fmt.Errorf("workspace: world %q head revision %d is mirror_only (blob evicted): cannot load", worldID, rev.ID)
   }
   ```
   with `return nil, notRematerializable(worldID, rev.ID, "load")`. Keep comment
   step 4 but drop the `// G4:` line. The guard stays **before** `blobs().Get` so the
   typed error is what callers see (not an opaque blobstore miss).

3. **Site 2 — `node_reload.go` `headBlobBytes`.** Replace the analogous block with
   `return nil, notRematerializable(worldID, rev.ID, "reload")`. The guard already
   sits before the `writeDropFileAtomic` + `Reload` in `ReloadNode`, so the existing
   ordering proof (no drop file written, no RELOAD sent on refusal) is preserved —
   `ReloadNode` wraps this with `%w` (`node_reload.go:79`), so `errors.Is` propagates
   to the public `ReloadNode` return. Verify the wrap chain stays `%w` end-to-end.

4. **Reuse, do not reinvent:** the `IsNotFound`/sentinel + `errors.Is` pattern already
   in `eviction.go:140-143` is the model. Do not add an `Is*` helper func unless a
   caller needs it — `errors.Is(err, ErrNotRematerializable)` is the idiom here.

## Risks / seams

- **Shared contract (downstream):** `ErrNotRematerializable` is the name E1 will
  `errors.Is` against to render a clean Starlark error. Get the **name and package**
  (`workspace.ErrNotRematerializable`) right now — renaming later breaks E1. This is
  the fan-out edge: G4 → E1 (the last gate on the E-path).
- **`%w` chain must be unbroken:** every wrap from the guard up to the public method
  (`Load`, `OpenWorld`, `ReloadNode`) must use `%w`, or `errors.Is` at the boundary
  fails silently and the typed refusal degrades to a plain error. `ReloadNode` wraps
  `headBlobBytes` with `%w` at line 79 — preserve that; do not switch any wrap to `%v`.
- **Guard ordering is load-bearing:** the `blob_present` check must stay **before**
  `blobs().Get` and before any side effect (the drop-file write in ReloadNode). Moving
  it after would (a) leak an opaque blobstore error instead of the sentinel and (b)
  break the "no drop file on refusal" invariant the existing test asserts.
- **Do NOT guard `reparseCommitted` (commit.go:167)** or add a guard in the F1 transfer
  path — both read blobs that are full-by-construction; a spurious guard there adds a
  redundant `RevisionByID` round-trip and risks a false refusal on the commit hot path.
- **No mirror→blob fallback:** the swap must remain a pure refusal. Do not "helpfully"
  attempt to re-derive bytes from the mirror — that is structurally forbidden by the spec.
- **Head is never normally evictable**, so these guards are defense-in-depth; the
  existing comments saying "a well-formed world's head is never evictable" stay true.
  Keep them — they explain *why* the guard is forward-compat, not dead code.

## Tier

**Tier: medium — exec=sonnet, review=opus.** Signals: it is a **shared contract**
(`ErrNotRematerializable` is the exact symbol E1 `errors.Is` matches; **fan-out** = 1
direct dependent E1 on the critical path) and a **subtle invariant** (correctness is
the unbroken `%w` chain — a passing-looking test that checks only `err != nil` would
*miss* a degraded-to-plain error; the tightened `errors.Is` assertions are what catch
it). It is otherwise additive/pattern-following over code that already works (two
pre-marked one-line swaps + a sentinel), so not high. Per the asymmetry rule, a cheap
executor follows the strong plan; a strong reviewer verifies the `%w` chain, the
name/package of the sentinel, and that no spurious guard was added to `reparseCommitted`
or the transfer path. No perf signal (no hot-path/unbounded-scan/N+1/tight-loop change;
the guards run once per Load/Reload and add no round-trips), so **no perf flag**.

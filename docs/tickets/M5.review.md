## Verified checks
1. **NodeIndex (logical) vs NodeRowIndex (slot) invariant — load-bearing.**
   `nodeByIndex` keys the memo map by the logical `node_index` value (read via
   `elementAttrs["node_index"].fieldIndex`), and stores the array ordinal as the
   handle's `index`. MUTATION TEST: rebuilt the package with a slot-based variant
   (key by `int64(ordinal)` instead of the logical node_index) →
   `TestSynapseSourceTargetByNodeIndex` FAILS with "no node with node_index 99".
   The test genuinely cannot pass a slot-based impl. (Fixture: NodeIndex 99/200 at
   slots 0/1 — the two deliberately differ.)
2. **PERF: nodeByIndex built ONCE (O(N)), not per call (O(N²)).** Build is guarded
   by `nodeByIndexOnce.Do(buildNodeByIndexMap)`; the only product-code trigger is
   inside `nodeByIndex` itself. `navSynapseEndpoint` calls the memoized lookup;
   no path resets the Once or rebuilds. EMPIRICAL PROBE: instrumented the build with
   a counter — 6 source/target lookups across a brain's synapses triggered exactly
   1 build. Per-element rebuild trap avoided; iterating a brain is O(N) total.
   (`navNodeSynapses` scans the entity's synapses per call, which the plan accepts;
   brains are small/bounded and there is no growing-data N+1 here.)
3. **Loud miss.** Dangling `node_in`/`node_out` → `fmt.Errorf("synapse.%s: no node
   with node_index %d")`, never silent None. MUTATION TEST: replaced the error with
   `starlark.None, nil` → `TestSynapseSourceMissIsLoud` FAILS. Test is real.
4. **inputs/outputs.** Return `*starlark.List` of synapse `*ArrayElement` handles
   (NOT a stale-filter `*ElementCollection`), each carrying its true array ordinal.
   `TestNodeInputsOutputs` checks length + ordinal set + endpoint equality against a
   brute-force scan. Kind-generic dispatch (egg AND bibite) via
   `subCollectionRegistry()[e.kind]`, exercised by `TestEggBrainNavigation` (round-trip
   source.outputs / target.inputs on an egg brain).
5. **Scope.** No `.where` added on the navigation result (deferred to §4 per plan).
   `api/web/app.js` NOT touched. `AttrNames` gates source/target to synapse elements
   and inputs/outputs to node elements; stomach answers none (`TestNavigationAttrNames`).
   `Attr` returns `(nil, nil)` for the wrong-kind case, matching the existing
   unknown-attr fall-through (Starlark reports a clean AttrError). No column-name
   collision (synapse/node specs have no `source`/`target`/`inputs`/`outputs` column).

## Non-negotiables (verified in code)
1. SINGLE-SAVE BYTE-IDENTICAL — PASS. Pre-E3 `whereClause` emitted
   `"WHERE " + quoteIdent(identity) + ".save_id = ?"` / `[ls.saveID]`. `workingScope.scopeClause`
   returns exactly `quoteIdent(identity)+".save_id = ?"` / `[saveID]`; `whereClause` prepends
   `"WHERE "`. Identical string + args. `TestWorkingScopeClauseByteIdentical` asserts it; the whole
   thebibites suite stays green; all mutation builders (`bulkSetQuery`, `bulkSetExprQuery`,
   `matchingEntryNames`) pass `ls.workingScope()` explicitly, so mutation is single-save by
   construction.
2. CROSS-WORLD ISOLATION — PASS (headline). `TestSpanningTwoWorlds/isolation` asserts each
   world's spanning count == THAT world's own count AND workspace == A+B, across bibites/eggs/
   pellets, with a fixtures-must-differ guard. Ground truth comes from an INDEPENDENT path
   (`HistoryQuery` over `world_saves`) cross-validating the spanning `mirror_saves` scope.
   Mutation-revert proves the test catches a silent SUM.
3. SCOPING BY CONSTRUCTION — PASS. The IN-subquery comes from the catalog
   (`save_id IN (SELECT save_id FROM mirror_saves [WHERE world_id=?])`); confirmed by dumping the
   generated SQL: a plain count needs no JOIN and no caller predicate; `where(world_id)` rewrites
   the bare token to `mirror_saves.world_id` and pulls the LEFT JOIN only when a catalog column is
   referenced (`catalogJoinIfNeeded`). The script never writes save_id or a JOIN
   (`TestSpanningTwoWorlds/where_world_id` + the Starlark no-JOIN program).
4. READ-ONLY / AGGREGATE-ONLY / NON-ITERABLE — PASS. Three independent gates:
   `Attr`/`AttrNames` hide set/set_expr/delete on a read-only scope; `rejectMutation`
   belt-and-suspenders if a builtin is reached directly; `resolveOrPanic` makes Len/Iterate
   panic loudly (aggregate-only), `EntryNames` rejects spanning transfer, and `whereBuiltin`
   skips eager entry-name resolution for spanning. Verified through Starlark
   (`TestSpanningTwoWorlds/mutation_rejected`) and white-box
   (`TestSpanningMutationBuiltinRejectsLoudly` asserts stagedOps==0;
   `TestSpanningCollectionNotIterable`).
5. world_id/sim_time FRIENDLY COLS + eggs/pellets parity — PASS. `catalogColumns` is exactly the
   two catalog DDL columns (no parallel entity allowlist); entity columns keep deriving from
   `attrRegistry()`. The `pellet` analytics kind is additive (`entityTables["pellet"]={"pellets"}`,
   columns from generated metadata via the shared `tableScalarSpecs`/pelletOverrides). It is
   aggregate-only by construction: `locatorSelect` and `entityLocatorRef` return "unknown entity
   kind" for pellet, so no mutation/delete/transfer/iteration path can reach it; the read-only
   scope blocks mutation regardless. The in-memory `save.pellets` surface is untouched. Isolation +
   group_by('world_id') run for all three kinds.
6. PERF-SHAPE — PASS. `spanningReader` runs `refreshMirrorCatalog` once per call under `w.mu`
   (released before the aggregate), fingerprint-gated (cache hit when unchanged). `where()`/`count()`
   carry the scope forward and run ONE aggregate SELECT — no per-revision loop, no rebuild per
   `.where()`. `TestSpanningTwoWorlds/perf_shape` asserts ZERO extra `rebuildCount` and ZERO extra
   `scenesReadCount` across 5 repeated reads (count + where) on an unchanged workspace.

## Tier

**Tier: high ‚Äî DSL-chain ROOT with fan-out 4 (M1, M5, M7 direct deps; M2
transitive), plus a shared-contract change (the keyed-lookup / miss / handle-shape
the dependents read against) and a breaking surface removal (`gene()`).** A silent
miss-contract bug here propagates to four downstream tickets. ‚Üí `--model-exec opus
--model-review opus`.

No perf flag. M4 touches no hot path, no unbounded scan, no N+1: it converges
per-entity binding shapes (handle reads, a map `.get`, deleting one builtin). All
reads stay on the existing in-memory `geneSet`/registry paths; nothing new runs
per-row or per-revision.

## Goal

Make the five "child rows of an entity/scope" surfaces (genes / settings / brains /
stomach / zones) teach **one** read+lookup+miss contract, and delete the redundant
`gene("X")` point-read so the gene surface is a single `GeneCollection`. "Done" =
every named-keyed child surface answers `["X"]` (loud `KeyError` on miss) and
`.get("X", default=None)` (tolerant) with the same case-fold rules M3 landed; every
element is a **handle** read by column attributes (no value-direct gene read leaking
the bare scalar through `["X"]`'s neighbours ‚Äî see Scope for the precise, narrow cut);
`b.gene(...)` is gone everywhere it was registered; and the hand-typed autocomplete
member sets reflect the new surface. This is the **read/lookup/miss convergence**;
the *write*-idiom convergence (attr-assign on handles, structural append on brains,
NULL/bulk writes) and the *spanning* lift are explicitly deferred to M7/M5/M1 (see
Scope decisions) ‚Äî M4 makes the contract those tickets build on stable.

**IN scope for M4 (the read / keyed-lookup / miss / handle contract):**

1. **Remove `gene("X")` entirely** (¬ß4 "Remove `gene("X")`", lines 226-237). Delete
   the `gene` builtin registration, `geneBuiltin` impl, and the `gene` `AttrNames()`
   entry from `entity.go`. The tolerant read it provided becomes `genes.get("X")`.
2. **Add `GeneCollection.get(key, default=None)`** ‚Äî Starlark-dict-exact signature
   (¬ß4 open question ".get signature", line 268). Loud on a fold-collision (reuses
   `foldLookup`'s error), tolerant on a genuine miss (returns `default`). This is the
   single sanctioned tolerant gene read after `gene()` is gone.
3. **Add `SettingScope.get(key, default=None)`** with the identical signature/semantics,
   so the two name-keyed surfaces (genes, settings) share one tolerant-read idiom ‚Äî the
   ¬ß4 convergence rule "keyed lookup (`["X"]`/`.get`) only where the element has a name
   key" applied uniformly to both name-keyed surfaces.
4. **Make `SettingScope` iterable + `len()`** so `for s in scope` / `len(scope)` work
   the way `for g in b.genes` / `len(b.genes)` already do ‚Äî closes the "iterate only on
   genes" gap for the *other* name-keyed surface and gives M2's cheatlist one shape to
   document. (Genes already implement `Iterable`/`Sequence`/`Len`; settings do not.)
5. **Settings iteration yields the `Setting` handle** (not a bare value) ‚Äî already the
   shape `scope["X"]` returns, so iteration and subscript agree. This makes "every
   element is a handle" true for settings without touching genes' write path.
6. **Regenerate the hand-typed autocomplete member sets** in `api/web/app.js`
   (`AC_TYPE_MEMBERS`, `AC_RESULT`, and the flat name pool near line 1583): drop
   `gene`, ensure `genes`/`get` are present, add a `setting_scope` member set
   (`get` + iteration), so the editor stops advertising the removed builtin and starts
   advertising `.get`. **This is hand-edit-to-match, NOT a generation pipeline** ‚Äî
   ¬ß2/M2 owns turning `AC_TYPE_MEMBERS` into a generated artifact; M4 only keeps the
   hand copy honest (memory: "derive, don't duplicate" is M2's mandate, not M4's).

**Resolved open questions (decisions M4 commits, for the dependents to rely on):**
- *Handle vs value for scalars (line 252):* keep genes' **value-direct** read through
  `b.genes["X"]` / `.get("X")` (returns the bare scalar) ‚Äî the `Gene` **handle** with
  `.value`/`.name`/`.type` already exists and is what *iteration* yields. We do NOT
  force `g.value` on subscript in M4: that is a breaking read-shape change with no
  dependent that needs it, and the spec marks it "acceptable pre-v1" but not required.
  Settings keep handle-on-subscript (their existing shape). Net contract the
  dependents see: **subscript/`.get` on a name-keyed mapping returns whatever that
  mapping has always returned; iteration always yields a handle.** Recorded so M2's
  cheatlist documents one rule, not a pending flip.
- *`.get` signature (line 268):* `get(key, default=None)`, default defaults to
  `starlark.None`, matching Starlark dict.
- *Egg parity (line 269):* `gene()` removal and `genes.get` apply to **both** bibite
  and egg (the `Entity` type carries both kinds; the gene table differs only via
  `geneTable(kind)`). Confirmed no third caller of the point read exists (grep below).
- *Case-folding (line 270):* `.get`/`["X"]` inherit M3's `foldLookup` verbatim

## Risks / seams (what the executor must not break)

- **The miss asymmetry is the whole point ‚Äî keep it.** `["X"]` MUST stay loud
  (`Get` returns `found=false` ‚Üí Starlark raises `KeyError`); `.get` MUST be the only
  tolerant read. A reviewer-catchable failure mode is making `.get` raise on miss
  (defeats it) or making `["X"]` return `None` (silently reintroduces the bug ¬ß3/¬ß4
  removed). The new test asserts both directions.
- **Collision must stay loud through `.get`.** `.get` returns `default` only on a
  genuine `!found` from `foldLookup`; a fold *collision* returns `err`, which `.get`
  must propagate ‚Äî NOT swallow into the default. Easy to get wrong by treating "no
  single match" as "miss".
- **Default-arg defaults to `None`, not zero/empty.** `UnpackArgs("key", &key,
  "default?", &dflt)` with `dflt = starlark.None` before unpack. Match Starlark dict
  exactly so M2's cheatlist can describe one `.get`.
- **Both kinds (bibite + egg).** The `gene()` removal and `.get` addition apply through
  the shared `Entity` type ‚Äî verify the egg path (egg_genes) still reads via
  `genes`/`.get`. No egg-specific code remains referencing the point read (grep
  confirmed only bibite/egg share `Entity`).
- **Settings iteration order must be deterministic.** Tests and the cheatlist depend on
  a stable order; if no stored order exists for a scope, sort by setting name. Do not
  rely on Go map iteration order.
- **Do not touch the write paths.** `GeneCollection.SetKey`, `Setting.setBuiltin`,
  zones, subcollection append/delete stay byte-identical ‚Äî M7 owns the write
  convergence and will conflict if M4 pre-empts it. M4 adds read/lookup surface only.
- **app.js is hand-edited to match, not generated.** Keep the diff minimal and aligned
  to Go `AttrNames`; a generation pipeline here would collide with M2.
- **Build-artifact hygiene** (memory: orchestrator gotcha): do not build binaries into
  the worktree/repo root; `go test ./script/...` only. No `go build -o`.

# D2 — simctl passthrough: stop / resume / info + NodeState

Source: `docs/workspace_plan.md` §D2 (Track D). Depends on D1 (node lifecycle +
active set), merged into the worktree.

## Tier

Tier: **medium** — exec `sonnet`, review `opus`. Signals: thin passthrough over
the already-working, already-tested `simctl.Client` (clears every high signal —
no schema/migration/transaction/crash-safety/security/novel-algorithm work) and
additive (new file + new methods, existing behavior unchanged), so the *generator*
is cheap; but it (a) defines a new exported `NodeState` shape that **D3** (reload
path) and **E1** (`node.info()` Starlark) consume — a downstream-consumed contract
— and (b) carries a deadlock seam (must not block on an IPC `Request` while holding
`w.mu`). A green test can pass while still holding the lock across IPC, and a wrong
`NodeState` shape propagates to D3. That earns an `opus` reviewer; it does not earn
an `opus` executor (it follows the established simctl pattern, not novel logic).

## Goal

Add node sim-control passthrough methods to `workspace.Workspace` so a caller can,
by **logical node id**, pause (`NodeStop`), run at a time scale (`NodeResume`), and
read live telemetry (`NodeInfo`) of the game node — each by constructing
`simctl.New(rt)` over the active runtime and calling the matching `simctl.Client`
method — plus a `NodeState(nodeID)` accessor that combines the runtime's process/
connection state (`noderuntime.Runtime.State()`) with the last `ipc.InfoResult`.
"Done" = these methods exist in a new `workspace/node_control.go`, are method-scoped
and additive (no struct-field changes, no edits to `node.go`/`workspace.go`), look
the runtime up via the existing active set under `w.mu` but perform the blocking IPC
call **without** holding `w.mu`, surface clean errors for unknown / disconnected
nodes, and a fake-compat-server test proves STOP/RESUME/INFO round-trip plus the
combined-state and error paths. No new IPC wire verbs; no schema changes.

## Approach

1. **Shared lookup helper (in `node_control.go`).** All four methods need the
   active runtime by logical node id; reuse D1's accessor `w.Node(nodeID)`
   (`node.go:188`, already locks `w.mu` internally and returns `(*noderuntime.Runtime,
   bool)`). Do **not** re-implement a locked map read — call `w.Node`. On `!ok`
   return a clean error (`fmt.Errorf("workspace: <Method>: unknown node %q", nodeID)`).
   This keeps the lock critical section to the map read only; the subsequent blocking
   IPC call happens with no lock held (deadlock seam — see Risks).

2. **`NodeInfo` / `NodeStop` / `NodeResume`** — each:
   - nil-guard the workspace (`if w == nil { return zero, fmt.Errorf(...) }`), as
     every D1 method does (`node.go:64`).
   - `rt, ok := w.Node(nodeID)`; `!ok` → error.
   - `c := simctl.New(rt)` — `*noderuntime.Runtime` already satisfies
     `simctl.Requester` (`simctl.go:23`; the doc comment on the interface names
     `noderuntime.Runtime.Request` explicitly, and `Runtime.Request` exists at
     `runtime.go:239`). No wrapper needed.
   - Delegate: `c.Info(ctx)` (`simctl.go:52`), `c.Stop(ctx)` (`simctl.go:37`),
     `c.Resume(ctx, timeScale)` (`simctl.go:44`). Return the typed result + error,
     wrapping the error with method context (`fmt.Errorf("workspace: NodeInfo %q:
     %w", nodeID, err)`).
   - **Disconnected node:** when the runtime has no compat session, `Runtime.Request`
     returns `noderuntime.ErrNoSession` (`runtime.go:243-245`); it flows out wrapped.
     Do not special-case it beyond wrapping — surfacing the typed error is correct
     and lets E1 render a clean Starlark error later.
   - **`NodeResume` validation:** the wire contract requires `timeScale > 0` (the DLL
     rejects `<= 0`; see the fake's RESUME handler and `simctl.go:44` doc). Let the
     server enforce it (matches `simctl_test.go` TestResume semantics) — do NOT add a
     client-side guard, to keep this a pure passthrough and avoid diverging from the
     simctl client's contract. (If a guard is wanted later it belongs in
     `simctl.Resume`, not here.)

3. **`NodeState`** — the combined snapshot the plan calls for ("Surface a `NodeState`
   combining `Runtime.State()` + last `InfoResult`", §D2):
   - `rt, ok := w.Node(nodeID)`; `!ok` → error.
   - Capture `rtState := rt.State()` (`runtime.go:142`; returns `noderuntime.State`
     with PID/process info/`Connected`). This is cheap and lock-free w.r.t. `w.mu`.
   - If `rt.Connected()` (`runtime.go:176`), call `simctl.New(rt).Info(ctx)` to fetch
     the latest `ipc.InfoResult` and set `Info: &info`. If the INFO call fails,
     **return the error** (do not silently drop telemetry) — a `NodeState` that
     claims connected-but-no-info would mislead D3/E1. If the runtime is not
     connected, set `Info: nil` and `Connected: false` and return success (a started-
     but-not-yet-connected node is a legitimate state, not an error).
   - **`NodeState` shape (load-bearing for D3/E1):**
     ```go
     type NodeState struct {
         Runtime   noderuntime.State // PID, process info, compat addr, connected
         Connected bool              // == Runtime.Connected; hoisted for callers
         Info      *ipc.InfoResult   // last INFO telemetry; nil when disconnected
     }
     ```
     `Info` is a pointer so "no telemetry" is representable distinctly from a
     zero-valued `InfoResult` (which `Paused:false, TPS:0` would otherwise look like
     a real reading). This mirrors `ipc.InfoResult.LastAutosave` being a pointer for
     the same reason (`commands.go:56`). Keep `Connected` hoisted so callers do not
     have to reach into `Runtime`.

4. **No `w.mu` around IPC.** `w.Node`/`rt.State`/`rt.Connected` each take and release
   the relevant lock internally and return quickly. The `simctl` `Request` round-trip
   (blocking, network) runs after those returns, with **no workspace or runtime write
   lock held by D2 code**. Confirm by inspection that no D2 method holds `w.mu` across
   a `c.Info/Stop/Resume` call.

## Tests — `workspace/node_control_test.go`, package `workspace` (white-box)

Reuse the **simctl fake-server pattern** (`simctl/simctl_test.go:19-96`): an
in-process `fakeSim` over `net.Pipe` that answers STOP/RESUME/INFO/RELOAD with the
exact envelope contract. The seam that lets D2's *node-id-keyed* methods reach a fake
(without a real game process) is `noderuntime.Wrap` (`runtime.go:118`):
`noderuntime.Wrap(nodeID, runID, nil /*no process*/, ipc.NewSession(clientConn, nil))`
builds a Runtime whose `Request` is backed by the pipe, and whose `Connected()` is
true (session != nil). Because the test is **white-box** (package `workspace`), it can
insert that runtime directly into the unexported active set: `ws.nodes[nodeID] = rt`
under `ws.mu` (or via a tiny in-test helper). This avoids `StartNode`'s real-process
requirement and keeps these tests fast and OS-independent (no `/bin/sleep` skip).

Test helper to add to this file:
- `func newFakeNode(t, ws, nodeID string) (*noderuntime.Runtime, *fakeSim)` —
  `cConn, sConn := net.Pipe()`; build the fake server on `sConn` (copy/adapt the
  `fakeSim` serve/handle from `simctl_test.go` — it is unexported there, so D2's test
  file needs its own copy of the minimal fake, or a trimmed version answering only
  STOP/RESUME/INFO); `rt := noderuntime.Wrap(nodeID, "run", nil, ipc.NewSession(cConn,
  nil))`; register `rt` in `ws.nodes[nodeID]` under `ws.mu`; `t.Cleanup` closes both
  sessions. Create the `ws` via `Create(ctx, t.TempDir(), owner, name)` (a world row
  is not required for these passthrough methods, but `createTestWorkspaceAndWorld`
  from `node_test.go` is available if a bound world is wanted).

Cases:
1. **NodeInfo round-trip.** Register a fake node; `NodeInfo(ctx, id)` returns the
   fake's `InfoResult` (TPS/RealTPS/Paused/SimTime and `LastAutosave.Name`), mirroring
   `simctl_test.go` TestInfo assertions.
2. **NodeStop round-trip.** `NodeStop(ctx, id)` returns `PreviousTimeScale == 3.5`
   (the fake's STOP reply); assert the fake saw command `ipc.CommandStop`.
3. **NodeResume round-trip.** `NodeResume(ctx, id, 4.25)` returns `TimeScale == 4.25`;
   assert the server received `TimeScale == 4.25`. Also assert `NodeResume(ctx, id, 0)`
   returns an error (the fake rejects `<= 0`, proving the contract is enforced
   server-side end-to-end).
4. **NodeState (connected).** `NodeState(ctx, id)` returns `Connected == true`,
   `Info != nil` with the fake's telemetry, and `Runtime` carrying the node id; assert
   `Runtime.Connected == true`.
5. **NodeState (disconnected).** Register a runtime with **no session**
   (`noderuntime.Wrap(id, "run", nil, nil)`); `NodeState(ctx, id)` returns success
   with `Connected == false` and `Info == nil` (a not-yet-connected node is not an
   error). And `NodeInfo`/`NodeStop`/`NodeResume` on that same node return an error
   that wraps `noderuntime.ErrNoSession` (`errors.Is`).
6. **Unknown node.** `NodeInfo`/`NodeStop`/`NodeResume`/`NodeState` on an
   unregistered logical id each return an error (mirrors
   `TestStartNode_UnknownNodeStop`, `node_test.go:371`).

Run: `go test ./workspace/...` then `go test ./...` before advancing.

## Risks / seams the executor must not break

- **No `w.mu` held across the IPC request (deadlock / serialization seam).** The
  blocking `simctl` round-trip must run with no workspace lock held. Use `w.Node`
  (which locks/unlocks internally) for the lookup, then call the client. Holding
  `w.mu` across a network `Request` would serialize all workspace ops behind a slow
  game and risk deadlock with concurrent mutators (AddWorld/Commit in later tickets).
  A passing test will NOT catch this — verify by inspection.
- **Downstream-consumed `NodeState` shape (fan-out to D3 + E1).** D3's reload path
  and E1's `node.info()` Starlark binding read this struct. Keep `Info` a pointer
  (nil == no telemetry, distinct from a zero reading) and keep `Connected` hoisted.
  Changing the shape later forces edits in D3/E1, so get it right now; record the
  rationale (above) so a reviewer can confirm intent.
- **Pure passthrough — no client-side policy.** Do not validate `timeScale`, retry,
  or reinterpret results in D2; `simctl.Client` is the contract owner and is already
  tested. Adding logic here diverges the two call paths (raw `simctl` vs workspace)
  and is the easiest way to introduce a subtle behavior difference.
- **Disconnected vs error.** `NodeInfo/NodeStop/NodeResume` on a session-less runtime
  must surface `noderuntime.ErrNoSession` (wrapped, `errors.Is`-able). `NodeState`,
  by contrast, treats "not connected" as a **valid** state (Info nil, no error) —
  only a *failed INFO while connected* is an error. Do not conflate the two.
- **Method-scoped, additive, clean merge.** All D2 code lives in the new
  `node_control.go` / `node_control_test.go`. Do NOT add fields to the `Workspace`
  struct and do NOT edit `node.go` or `workspace.go` — D1 already wired the active
  set and `Close()` drain; D2 only reads the active set. Reuse `w.Node`, `w.mu`,
  `simctl.New`, and `noderuntime.Runtime.{State,Connected,Request}`.
- **No new IPC verbs / no DLL change / no schema change.** D2 only orchestrates the
  existing STOP/RESUME/INFO commands via the existing `simctl.Client`.

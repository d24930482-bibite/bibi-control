## Seam-by-seam (the spots a green test can miss)

1. **Non-reentrant mutex (OpenWorld must not nest w.mu into Load).** Traced by
   hand. `OpenWorld` does `w.mu.Lock()`; in the hit branch it `w.mu.Unlock()`
   before `return ls, nil`; in the miss branch it `w.mu.Unlock()` and then calls
   `w.Load(...)` with NO lock held. `Load` re-takes `w.mu` via
   `Lock()/defer Unlock()`. No nesting, no self-deadlock. The lazy path is
   exercised live by `TestOpenWorldLazyLoads` (a held lock would hang/time out).
   CORRECT.

2. **LoadInto contract (worldID, not blob sha256, as save_id).** `Load` calls
   `thebibites.LoadInto(tmpPath, worldID, w.duck())` — second arg is `worldID`
   (working_set.go:124-ish). Verified against `script/thebibites/loadedsave.go:146`
   (`LoadInto(path, worldID, db)`), and `openDB` (loadedsave.go:426-429)
   short-circuits on the injected non-nil handle and imports nothing. So the
   shared per-workspace DuckDB working partition C1 seeded under `save_id=worldID`
   is reused, not re-imported. No duplicate rows: `TestLoadReturnsHeadWorkingCopy`
   and `TestLoadTwoWorldsIndependent` assert `countBySaveID(save_archives, world.ID)
   == 1` (exactly one archive row — a double-seed would make it 2). CORRECT.

3. **Guards.**
   - nil `HeadRevisionID`: clean error `"world %q has no head revision"` before any
     deref of `*world.HeadRevisionID`. Pinned by `TestLoadWorldWithoutHeadFails`
     (bare `CreateWorld` leaves head NULL; a nil deref would panic the test).
   - `!rev.BlobPresent`: present as a plain error labeled with
     `// G4: replace with ErrNotRematerializable`. Not silently missing; correctly
     scoped as a C2 stub (G4 owns the typed error). CORRECT.

4. **Unload.** Body is `delete(w.worlds, worldID)` under the lock — no

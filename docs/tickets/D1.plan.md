# D1 — Node lifecycle: start / persist / bind + active set

Source: `docs/workspace_plan.md` §D1 (Track D). Depends on B2 (workspace skeleton, merged).

## Tier

Tier: **medium** — exec `sonnet`, review `opus`. Signals: fan-out (2 dependents,
D2 + D3, build on the handle/Spec shape and active set), a process-lifecycle
cleanup seam (process started but a later persist/bind step fails → must not strand
an OS process), and a binding invariant a green test can miss ("a world is bound to
at most one active node"; logical `nodeID` vs registry-row PK key discipline). It
clears `low` (not purely additive — it owns orphan-process cleanup and a uniqueness
invariant) but not `high` (no schema/migration/transaction/crash-safety/security
work; it orchestrates already-working `noderuntime.Start` + A2 store methods rather
than writing novel algorithmic logic).

## Goal

Add node-lifecycle methods to the `workspace.Workspace` so a caller can: start a
game node (launch the process via `noderuntime.Start`, optionally connecting the
compat session), persist its identity + binding as a `nodes` registry row, bind it
to a world, and track the live `*noderuntime.Runtime` in the in-memory active-node
set. Expose read accessors (`Nodes()`, `Node(id)`) over the active set and persisted
rows, and stop/kill methods that tear the process down and update the persisted
status. "Done" = these methods exist, are method-scoped and additive (clean merge
with C1 which edits `workspace/workspace.go` in parallel), the active-set map is
populated/drained correctly, the persisted node row carries identity + binding +
status, and a lifecycle test (start → persist → bind → stop, plus the bind-uniqueness
and orphan-cleanup seams) passes. No new IPC wire verbs; no schema changes.

## Approach

1. **Define `StartNodeSpec`** as the workspace-level input that wraps the lower
   `noderuntime.Spec` with the persistence/binding fields D1 owns:
   - `WorldID string` — the world to bind to (required; the plan locks "a node binds
     to a world").
   - `NodeID string` — logical node id (the active-set key and `nodes.node_id`). If
     empty, default it (e.g. `uuid.NewString()` via `github.com/google/uuid`, already
     a direct dep used in `revisionstore/store.go`). This is **distinct** from the
     `nodes` registry-row PK (`Node.ID`, assigned by `CreateNode`).
   - `RunID string` — run identity (passed to `noderuntime.Spec.RunID`, stored as
     `nodes.run_id`).
   - `Process ipc.ProcessSpec`, `CompatAddr string`, `ConnectOnStart bool`,
     `DialTimeout`/`DialInterval time.Duration`, `Codec ipc.Codec`, `DropPath string`
     — forwarded into `noderuntime.Spec` (all fields already exist on
     `noderuntime.Spec`, `runtime.go:23-45`) and persisted (`DropPath` →
     `nodes.drop_path`, for D3's ship-head path).

2. **`StartNode`** (acquire `w.mu` for the active-set + persist mutation; the workspace
   mutex is the documented single-writer guard, `workspace.go:41-45`):
   - Validate: non-nil workspace, `spec.WorldID != ""`. Resolve `nodeID` (default if
     empty). **Reject a duplicate active node**: if `nodeID` is already in `w.nodes`,
     return an error (don't start a second process under a live key).
   - **Enforce the one-node-per-world invariant**: scan the active set; if another
     live runtime is already bound to `spec.WorldID`, refuse. (Track the binding in
     the active set — see step 5 — so this check is cheap and doesn't require a DB
     round-trip.) The plan locks "A world is bound to at most one active node at a
     time" (§Core model).
   - Build a `noderuntime.Spec` from the spec fields and call `noderuntime.Start(ctx,
     ns)` (`runtime.go:85`). `Start` already optionally dials the compat session when
     `CompatAddr != "" && ConnectOnStart`, and kills the process if the dial fails —
     reuse that; do not re-implement connect.
   - **Persist after the process is up**: `w.store().CreateNode(ctx,
     revisionstore.NodeInput{WorkspaceID: w.id, WorldID: spec.WorldID, NodeID: nodeID,
     RunID: spec.RunID, Status: "running", CompatAddr: spec.CompatAddr, DropPath:
     spec.DropPath})` (`store.go:655`). `CreateNode` persists the FK binding directly
     (the `world_id` column), so a separate `BindNode` call is **not** required on the
     start path — `BindNode` (`store.go:730`) is for later rebinding. Cite both but
     use `CreateNode` for the initial bind.
   - **Orphan-process cleanup seam (the medium-risk point):** if `CreateNode` fails
     after `Start` succeeded, the OS process is already running. Kill it
     (`rt.Kill()` / `rt.Close()`) before returning the error so a failed `StartNode`
     leaks no process — mirror the cleanup discipline `noderuntime.Start` itself uses
     when its own connect step fails (`runtime.go:110`).
   - On success: store `rt` in `w.nodes[nodeID]` (and its bound world — step 5),
     return `(rt, node, nil)`.

3. **`Nodes()` / `Node(id)`** — read accessors over the active set under `w.mu`.
   `Nodes()` returns a slice snapshot (stable, copied out of the map so callers can't
   mutate it). `Node(id)` returns the runtime + an ok bool. Nil-safe.

4. **`PersistedNodes(ctx)`** — thin passthrough to
   `w.store().ListNodes(ctx, w.id)` (`store.go:697`); read-only, returns the
   persisted rows (active set is the live subset).

5. **Active-set value vs binding lookup.** B2 declares `nodes map[string]*noderuntime.Runtime`
   keyed by logical node id (`workspace.go:51`). The one-node-per-world check needs
   world→runtime. Two clean options — pick the lower-churn one:
   - (preferred) keep B2's `map[string]*noderuntime.Runtime` as-is and derive the
     bound world by reading the runtime's persisted row when needed, OR keep a tiny
     parallel `map[string]string` (nodeID→worldID) **declared in `node.go`** is not
     possible (struct field). So: track binding by querying within `StartNode` —
     before start, call `ListNodes`/scan active set and `GetNode` to find any node
     whose `world_id == spec.WorldID` with status `running`. Prefer the in-memory
     active set for the liveness check (a persisted-but-dead node must not block a new
     bind). Keep the per-world reverse index in-memory only if the executor finds the
     repeated lookup hot; default to scanning `w.nodes` + `GetNode`, which is correct
     and simplest. **Do not add a struct field to `Workspace` for this** — that edits
     `workspace.go` and collides with C1; keep D1's state in the existing `nodes` map.

6. **`StopNode` / `KillNode`** (acquire `w.mu`):
   - Look up `rt` in `w.nodes[nodeID]`; unknown id → error.
   - `StopNode`: `rt.Stop(ctx, opts)` (`runtime.go:294`) — graceful command + grace +
     kill fallback are already implemented there. `KillNode`: `rt.Kill()`
     (`runtime.go:281`).
   - After teardown, `rt.Close()` to release the compat session, remove the key from
     `w.nodes`, and update the persisted status via
     `w.store().SetNodeStatus(ctx, persistedID, "stopped")` (`store.go:756`).
     **Note the id mismatch:** `SetNodeStatus` keys on the `nodes` PK (`Node.ID`), not
     the logical `node_id`. So `StartNode` must remember the returned
     `revisionstore.Node.ID` to pass here — capture it. Simplest: on stop, resolve the
     PK by scanning `PersistedNodes` for the matching `NodeID`+`WorkspaceID`, or have
     the active-set value carry it. Since the map value is fixed by B2 as
     `*noderuntime.Runtime` (do not change it to avoid the `workspace.go` collision),
     resolve the PK via `PersistedNodes`/`ListNodes` filtered by logical `NodeID`. The
     executor must handle this id indirection explicitly — it is the easiest place to
     introduce a silent bug (updating the wrong/no row).

7. **`Close()` drain (the only `workspace.go` edit):** before closing handles, iterate
   `w.nodes`, `rt.Kill()` (best-effort, join errors) and `rt.Close()`, then clear the
   map. Keep registry status updates out of `Close` (Close may run after the registry
   handle is already being torn down; a best-effort process kill is the contract).

## Tests — `workspace/node_test.go`, package `workspace` (white-box)

Use a real short-lived OS process so no compat server is needed (D2 owns the fake
compat server). A portable long-lived command: `ipc.ProcessSpec{Path: "/bin/sleep",
Args: []string{"60"}}` with `ConnectOnStart: false` and empty `CompatAddr`
(`noderuntime.Start` then only launches the process — `runtime.go:99`). Guard with a
`if _, err := os.Stat("/bin/sleep"); err != nil { t.Skip(...) }` for non-Linux CI.

Cases:
1. **Start → persist → bind → active set.** `Create` a workspace, `CreateWorld`
   (via `w.store().CreateWorld`) to get a real `WorldID`, `StartNode`. Assert: returned
   `revisionstore.Node` has `WorldID == world.ID`, `Status == "running"`,
   `NodeID`/`RunID` as supplied; `w.Node(nodeID)` returns the runtime with ok=true;
   `w.Nodes()` has length 1; `PersistedNodes` returns one row whose `world_id` matches.
   Then `StopNode` (or `KillNode`) and assert the runtime left the active set
   (`Node(nodeID)` ok=false), `w.Nodes()` is empty, and the persisted row's status is
   `"stopped"`.
2. **One-node-per-world invariant.** `StartNode` twice for the same `WorldID` (distinct
   logical node ids) → second call returns an error and starts no second process
   (`w.Nodes()` length stays 1). Clean up the first.
3. **Duplicate active node id.** `StartNode` twice with the same explicit `NodeID` →
   second returns an error.
4. **Orphan cleanup on persist failure (seam).** Hardest to force without a stub; if a
   clean injection point exists (e.g. closing the registry before `StartNode` so
   `CreateNode` errors) assert `StartNode` returns an error AND the process it started
   is not left running (the runtime's `State()`/`PID` reflects a killed process, or
   `w.Nodes()` is empty). If forcing this cleanly proves brittle, cover the kill path
   by asserting `Close()` drains a live node (start a node, `Close`, assert the process
   is killed via `rt.State()` / a short `rt.Wait`).
5. **Unknown node.** `StopNode`/`KillNode` on an unknown logical id returns an error.

Run: `go test ./workspace/...` then `go test ./...` before advancing the executor's
work.

## Risks / seams the executor must not break

- **Merge cleanliness with C1 (parallel).** C1 appends world methods to
  `workspace/workspace.go`. Put ALL D1 methods + `StartNodeSpec` in a new
  `workspace/node.go`. The only `workspace.go` change is the `Close()` drain — keep it
  to a few lines inside the existing function body. **Do not add fields to the
  `Workspace` struct** (that line is in `workspace.go` and would collide). Reuse the
  existing `nodes`/`worlds` maps and `mu`.
- **Two distinct ids.** Logical `node_id` (active-set key, `NodeInput.NodeID`) vs the
  `nodes` row PK (`Node.ID`). `BindNode`/`SetNodeStatus` key on the **PK**;
  `GetNode(id)` takes the PK. The active set is keyed by the **logical** id. Track
  the PK from `CreateNode`'s return so status updates hit the right row — getting this
  wrong silently no-ops the status update.
- **Orphan process on partial failure.** A started process is an OS resource; any error
  after `noderuntime.Start` succeeds must kill it before returning. This is the
  medium-risk correctness point — `noderuntime` will not clean up a process the
  workspace started but failed to register.
- **Liveness vs persistence for the per-world check.** The uniqueness invariant is
  about *live* nodes (the active set), not stale persisted rows. A node that crashed
  and left a `running` row in SQLite must not block a new bind — base the check on the
  in-memory active set.
- **Mutex discipline / single writer.** All active-set mutation + persistence runs
  under `w.mu` (the documented single-writer guard). Do not hold `w.mu` across the
  blocking `noderuntime.Start` dial in a way that deadlocks other ops — if
  `ConnectOnStart` with a long `DialTimeout` is used, consider releasing the lock
  around the blocking start and re-acquiring to mutate the map, while still
  serializing the uniqueness check + map write. Keep the critical section correct:
  re-check the duplicate/uniqueness condition after re-acquiring if the lock was
  dropped.
- **No new IPC verbs / no schema change.** D1 only orchestrates `noderuntime` +
  existing A2 store methods. The `nodes` table and all CRUD already exist.
- **`Close()` is best-effort and nil-safe.** The drain must not panic on an empty map
  (B2's tests already exercise empty-map Close) and must still close DuckDB/registry/
  blobs even if a kill errors (join errors, as the existing `Close` does).
